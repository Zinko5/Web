Usar git con github
# Primero es necesario tener git instalado y una cuenta en github

# Crear configuraciones globales para git en la computadora

# Identificarse git config --global user.name "Tu username"
git config --global user.name "Tu username" # Seleccionar vscode como el editor git config --global core.editor "code --wait" # Configurar el salto de linea para Linux git config --global core.autocrlf input # Si queremos vincular el directorio con github con HTTPS, crear una clave aqui, copiarla y guardarla, ya que no se podra volver a ver

# Si queremos vincular el directorio con github con SSH, crear una llave con: ssh-keygen # Copiar la llave copiando el resultado de: (Ctrl + shift + C para copiar en consola) cat ~/.ssh/NombredelaClaveCreada.pub # Pegar aqui y poner un titulo que identifique el equipo que creo la llave, guardar

# Ubicarnos en el directorio que deseamos usar como repositorio, ya sea seleccionando uno ya existente, creando uno nuevo o descargando uno de github (si lo hacemos, automaticamente se vinculara con HTTPS, pero podemos cambiarlo mas adelante) con: git clone https://direcciondelrepositorio.com # Si el directorio todavia no existe como repositorio en github, crearlo y poner en la consola git init
git branch -M main # Si queremos vincular el directorio con github con HTTPS: git remote add origin https://github.com/TuUsuariodeGithub/NombredelRepositorio.git
git push -u origin main # Si queremos vincular el directorio con github con SSH: git remote add origin git@github.com:TuUsuariodeGithub/NombredelRepositorio.git
git push -u origin main # Para cambiar el tipo de vinculacion, de HTTPS a SSH o viceversa git remote set-url origin Sintaxisdeltipodeseadocomoarriba.git # Ahora podemos usar los comando git add, git commit - m "mensaje indicando los cambios en el proyecto", git push y git pull en el directorio
